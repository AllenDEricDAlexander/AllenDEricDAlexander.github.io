Dell PowerEdge R730 完整使用指南：BIOS、iDRAC8、RAID 与 VMware ESXi 8 安装

Dell PowerEdge R730 是一台经典的 13 代双路 2U 机架式服务器。它的二手价格已经非常亲民，扩展性、内存容量、PCIe 插槽、远程管理和 RAID 能力却依然完整，因此很适合家庭实验室、虚拟化学习环境、私有云测试、Kubernetes 集群节点以及轻量 AI 推理环境。

不过，R730 毕竟是较早一代平台。它能够安装并运行 ESXi 8，不等于 Dell 或 Broadcom 正式认证这套组合。 如果用于生产环境，必须先检查 Broadcom Compatibility Guide、Dell 官方兼容性矩阵、CPU、网卡、阵列卡和磁盘控制器的支持情况；如果用于家庭实验室，则可以在充分备份、接受无厂商支持和潜在驱动风险的前提下尝试。

本文将从零开始介绍 R730 的硬件定位、BIOS 配置、PERC RAID、iDRAC8、Lifecycle Controller、VMware ESXi 基础概念，以及通过 iDRAC8 或 USB 安装 ESXi 8 的完整流程。

一、先说结论：R730 现在还值不值得用

对于家庭实验室、学习、测试和非关键业务，R730 仍然很有价值：

  • 双路 Intel Xeon E5-2600 v3/v4，核心数充足；
  • 最多 24 个 DDR4 DIMM 插槽，二手内存便宜；
  • 2U 机箱，PCIe 扩展空间比 1U 服务器宽裕；
  • 常见 PERC H730/H730P 阵列卡，硬件 RAID 管理成熟；
  • 自带 iDRAC8，可远程开关机、看硬件状态、挂载 ISO 和进入控制台；
  • 适合运行 ESXi、Proxmox VE、Linux、Kubernetes、NAS 测试环境和多台虚拟机。

它的短板也很明显：

  • 平台较老，单核性能、能效和待机功耗不如新服务器；
  • 风扇、硬盘和电源的噪声不适合普通卧室；
  • iDRAC8 已进入生命周期末期，固件不会继续获得常规功能更新；
  • ESXi 8 对整机、CPU、网卡和存储控制器的认证范围比 ESXi 7 更严格；
  • 第三方 PCIe 卡、非 Dell 硬盘或 GPU 可能导致风扇转速明显升高；
  • 二手机器的电池、缓存模块、风扇、背板和硬盘寿命需要重点检查。

因此，本文给出的建议是：

场景 建议
家庭实验室、虚拟化学习 可以安装 ESXi 8，接受非官方支持风险
Kubernetes、开发测试 可以，建议做好配置备份和替代节点
个人私有云 可以，关键数据另做独立备份
企业非关键测试环境 可以，但要验证网卡、PERC 和固件
企业核心生产业务 不建议把 R730 + ESXi 8 作为新部署方案
需要厂商正式支持 优先使用 Dell 兼容性矩阵明确支持的平台，或在 R730 上使用官方支持范围内的 ESXi 7.0 U3

二、Dell PowerEdge R730 硬件介绍

2.1 R730 的定位

Dell PowerEdge R730 是 13 代 PowerEdge 服务器，属于双路 2U 机架式机型。根据 Dell 官方手册，它支持：

  • 最多两颗 Intel Xeon E5-2600 v3 或 E5-2600 v4 处理器；
  • 最多 24 个内存 DIMM 插槽；
  • DDR4 ECC RDIMM/LRDIMM；
  • 最多 16 块 HDD 或 SSD，具体数量取决于前置背板版本；
  • 多个 PCIe 3.0 扩展插槽；
  • PERC 硬件 RAID 控制器；
  • 双热插拔冗余电源；
  • iDRAC8 与 Lifecycle Controller。

R730 的具体配置差异很大。购买二手机器时，不能只看“R730”三个字，还要核对 CPU、内存条规格、硬盘背板、阵列卡、网卡、PCIe Riser、电源功率和 iDRAC 许可证。

2.2 R730 与 R730xd 的区别

R730 更偏向计算与通用扩展，R730xd 更偏向高密度存储。

对比项 R730 R730xd
机箱 2U 2U
CPU 双路 E5-2600 v3/v4 双路 E5-2600 v3/v4
内存 最多 24 个 DIMM 最多 24 个 DIMM
主要定位 计算、虚拟化、PCIe 扩展 存储密集、虚拟化、软件定义存储
盘位 常见 8×2.5、16×2.5、8×3.5 常见 24×2.5 或 12×3.5，部分支持后置盘
PCIe 灵活性 通常更好 受存储背板和内部结构影响更多

具体盘位、背板和 PCIe 插槽布局必须按照服务标签查询，不能只凭外观判断。

2.3 双 CPU 与 PCIe 插槽的关系

R730 的部分 PCIe 插槽由第二颗 CPU 提供 PCIe 通道。如果机器只安装了一颗 CPU，某些插槽即使能够插入设备，也不会正常工作。

遇到网卡、HBA 或 GPU 插入后完全不识别时，依次检查：

  1. 当前插槽是否依赖 CPU2；
  2. CPU2 是否实际安装；
  3. 对应的 PCIe Riser 是否正确；
  4. BIOS 中是否禁用了该插槽；
  5. 设备是否需要 4GB 以上 MMIO；
  6. 电源功率和辅助供电线是否满足要求。

2.4 R730 上常见的管理与启动组件

很多新用户会把 BIOS、iDRAC、Lifecycle Controller、PERC 和 ESXi 混在一起。它们的职责如下：

三、首次拿到二手 R730：不要急着装系统

二手服务器最危险的操作不是装错系统，而是在不了解现有配置的情况下直接清空阵列或恢复默认设置。

3.1 记录服务标签和当前配置

先记录：

  • Service Tag；
  • Express Service Code；
  • CPU 型号和数量；
  • 内存总容量、频率和插槽分布；
  • 硬盘数量、容量、接口和健康状态；
  • PERC 型号；
  • 网卡型号和端口数量；
  • iDRAC8 许可证类型；
  • BIOS、iDRAC、PERC 和网卡固件版本；
  • 当前 RAID 级别和虚拟磁盘布局。

可以在以下位置查看：

  • 开机按 F2，进入 System Setup；
  • 开机按 F10，进入 Lifecycle Controller；
  • 登录 iDRAC8 Web；
  • 通过 Dell 支持网站输入 Service Tag。

3.2 检查硬件告警

在 iDRAC8 中重点检查：

  • System Event Log；
  • Lifecycle Log；
  • CPU 状态；
  • 内存 ECC 错误；
  • 风扇状态；
  • 电源冗余状态；
  • 物理磁盘状态；
  • PERC 缓存和电池状态；
  • 温度与入风口温度；
  • 是否存在 Predictive Failure、Foreign Configuration 或 Degraded。

不要一上来就清日志。正确做法是先导出或截图，再处理故障，确认无误后再清理旧记录。

3.3 检查固件基线

截至本文更新时间，R730 可参考的关键固件基线为：

组件 建议
BIOS Dell R630/R730/R730xd BIOS 2.19.0
iDRAC8 + Lifecycle Controller 2.86.86.86，也是 iDRAC8 最后一批计划内固件
PERC 按 Service Tag 下载对应控制器的最新固件
网卡 按具体 Broadcom/Intel 型号下载最新固件
背板、电源、CPLD 按 Dell 支持页和更新包说明处理

如果当前固件非常老，不要无脑跨越多个大版本。先阅读 Dell Release Notes；某些老版本需要分阶段更新。

一个较稳妥的更新顺序是：

  1. 导出 iDRAC 配置并确认服务器接入稳定电源；
  2. 更新 iDRAC8/Lifecycle Controller；
  3. 更新 BIOS；
  4. 更新 PERC/HBA；
  5. 更新网卡、背板、电源和其他设备固件；
  6. 完整关机断电一次，再重新上电检查硬件。

固件更新期间不要强制断电。双电源也救不了一只正在写 Flash 的手。

3.4 是否应该恢复 BIOS 默认值

对于刚买来的、无业务数据的实验室服务器，可以先记录当前设置，再加载 BIOS Default，然后按本文重新配置。

对于正在运行生产业务、来源不明或阵列结构不清楚的机器，不要直接恢复默认值，因为这可能改变：

  • Legacy BIOS/UEFI 启动模式；
  • 启动顺序；
  • SATA 模式；
  • 虚拟化选项；
  • SR-IOV；
  • PCIe 插槽状态；
  • 安全启动；
  • 串口重定向；
  • 电源与性能策略。

五、R730 BIOS 配置详解

下面以“通用虚拟化主机”为目标给出推荐配置。不同 BIOS 版本的菜单名称可能略有不同。

5.1 System Information

路径：

这里主要用于确认：

  • System Model Name；
  • BIOS Version；
  • Service Tag；
  • CPU 型号；
  • System Memory Size；
  • UEFI 版本；
  • CPLD 版本。

如果系统识别的内存容量少于实际安装容量，先检查内存插槽布局和硬件告警，不要急着安装 ESXi。

5.2 Memory Settings

路径：

通用虚拟化推荐：

设置项 推荐值 说明
System Memory Testing 首次验机开启，稳定后可按需关闭 每次开机测试会增加启动时间
Memory Operating Mode Optimizer Mode 通用性能最好
Node Interleaving Disabled 让 ESXi 正确识别 NUMA 节点
Patrol Scrub Enabled/默认 主动检测可纠正内存错误
Correctable Error Logging Enabled 便于发现 DIMM 老化

如果开启 Memory Mirroring、Spare Mode 或 Dell Fault Resilient Memory，会牺牲可用容量，只有明确需要可靠性特性时再启用。

5.3 Processor Settings

路径：

通用虚拟化推荐：

设置项 推荐值 说明
Logical Processor Enabled 开启超线程
Virtualization Technology Enabled ESXi 和嵌套虚拟化的基础
Execute Disable Enabled 安全功能
Hardware Prefetcher Enabled 通常保持默认
Adjacent Cache Line Prefetch Enabled/默认 通常保持默认
Dell Controlled Turbo Enabled 允许睿频

x2APIC Mode Enabled/默认 现代 ESXi 一般可正常使用

如果需要在虚拟机中继续运行 ESXi、KVM 或 Hyper-V，还要在虚拟机配置中开启嵌套虚拟化；仅在物理 BIOS 中打开 VT-x 并不够。

5.4 Boot Settings

路径：

对于全新安装 ESXi 8，推荐：

设置项 推荐值
Boot Mode UEFI
Boot Sequence 将 ESXi 系统盘放在首位
BIOS Boot Settings 不再使用则保持空或禁用无关项
UEFI Boot Settings 确认 PERC Virtual Disk 或目标磁盘存在

为什么推荐 UEFI：

  • 更适合现代 ESXi；
  • 支持 Secure Boot；
  • 对大容量启动盘更友好；
  • 后续升级和硬件兼容性通常更好。

如果原系统以 Legacy BIOS 模式安装，直接切换 UEFI 后可能无法启动。重装前必须确认当前系统的启动模式。

5.5 Integrated Devices

路径：

建议重点检查：

设置项 推荐值 使用场景
USB 3.0 Setting Enabled 使用 USB 3.0 安装介质或外设
Integrated Network Card Enabled 板载网卡
Embedded NIC1/NIC2 Enabled ESXi 管理与业务网络
SR-IOV Global Enable Disabled，按需开启 只有明确使用 SR-IOV 时开启
Memory Mapped I/O above 4GB GPU/大 BAR 设备时开启 GPU 直通、高端 PCIe 设备
Slot Disablement 确认目标插槽未禁用 网卡、HBA、GPU 不识别时检查
Internal USB Port 按需 使用内部 USB 时开启
Internal SD Card 不建议作为 ESXi 8 长期启动盘 SD 卡耐久性有限

GPU 直通额外建议

如果计划把 Tesla P4、P40、K80 或其他 PCIe 设备直通给虚拟机，通常需要：

  • Virtualization Technology：Enabled；
  • SR-IOV Global Enable：按设备和方案决定；
  • Memory Mapped I/O above 4GB：Enabled；
  • 确认插槽依赖的 CPU 已安装；
  • 确认 Riser、供电线和电源功率满足要求；
  • ESXi 中把设备标记为 Passthrough；
  • 大显存 GPU 可能需要在虚拟机高级参数中配置 64-bit MMIO。

GPU 直通属于另外一个专题，不能仅凭“ESXi 看见了显卡”就认为配置完成。

5.6 Serial Communication

如果没有串口控制台需求，可保持默认。数据中心常用串口重定向进行带外调试，家庭实验室一般不需要修改。

5.7 System Profile Settings

路径：

常见模式：

模式 适合场景 特点
Performance Per Watt (DAPC/OS) 通用虚拟化、长期运行 性能与功耗平衡
Performance 数据库、AI、低延迟、高负载 性能优先，功耗和噪声更高
Dense Configuration 高密度部署 更保守的散热和功耗策略

家庭实验室建议先使用 Performance Per Watt。如果发现 CPU 频率响应慢、延迟敏感或持续重负载，再切换 Performance 做对比测试。

不要为了“性能”盲目关闭所有节能选项。双路服务器在空闲时也会认真吃电，电表不会因为你在学习虚拟化就网开一面。

推荐：

  • 首次排障时可暂时关闭 Secure Boot；

六、PERC RAID 配置

6.1 先理解物理磁盘与虚拟磁盘

PERC 不会把每块硬盘直接交给 ESXi。常见硬件 RAID 模式下，需要先在 PERC 中把物理磁盘组成 Virtual Disk，ESXi 才能看到这个逻辑磁盘。

6.2 推荐 RAID 布局

方案 A：虚拟化主机的稳妥方案

  • 2 块企业级 SSD：RAID 1，安装 ESXi；
  • 4 块或更多数据盘：RAID 10，作为 VMFS Datastore；
  • 关键虚拟机再做独立备份。

优点是系统盘和数据盘互相隔离，后续维护清晰。

方案 B：硬盘较少的家庭实验室

  • 2 块 SSD：RAID 1；
  • ESXi 与 VMFS Datastore 共用同一个 Virtual Disk。

配置简单，但系统与数据故障域相同。

方案 C：容量优先

  • 6 块及以上容量盘：RAID 6；
  • 适合归档、镜像仓库和吞吐型负载；
  • 随机写性能和重建时间通常不如 RAID 10。

不建议在老旧大容量机械盘上新建承载关键业务的巨大 RAID 5。重建期间第二块盘再出问题，剧情就会从教程切换到灾难片。

vSAN、ZFS、Ceph 的特殊情况

如果准备使用 vSAN、ZFS 或 Ceph，不要直接套用传统硬件 RAID 方案：

  • vSAN 需要严格检查 Broadcom vSAN HCL；
  • ZFS 通常更适合 HBA/JBOD，把原始磁盘直接交给系统；
  • Ceph 通常也希望直接管理单盘；
  • 不建议把 ZFS 建在一个巨大硬件 RAID Virtual Disk 之上；
  • PERC 是否支持合适的 HBA/JBOD 模式，要看控制器型号和固件。

6.3 在 F2 中创建 RAID

操作路径：

操作步骤：

  1. 进入 Create Virtual Disk；
  2. 选择 RAID Level；
  3. 选择参与阵列的物理磁盘；
  4. 设置 Virtual Disk 名称；
  5. 设置容量；
  6. Stripe Size 没有明确测试依据时保持默认；
  7. Read Policy 可使用默认或 Adaptive Read Ahead；
  8. Write Policy 在缓存和电池正常时可使用 Write Back；
  9. 执行 Fast Initialization；
  10. 在 Controller Management 中确认启动虚拟磁盘。

创建或初始化 Virtual Disk 可能清除数据。执行前必须确认物理盘编号、槽位和备份。

6.4 Foreign Configuration 怎么处理

二手硬盘或更换阵列卡后，PERC 可能看到 Foreign Configuration。

  • 原阵列需要恢复：先研究原布局，通常应考虑 Import Foreign Configuration；
  • 所有数据都不要了：确认备份后再 Clear Foreign Configuration；
  • 不清楚来源：不要随便 Import，也不要随便 Clear。

6.5 Write Back 与缓存电池

只有在以下条件满足时才建议使用 Write Back：

  • PERC 缓存模块正常；
  • CacheVault/电池状态正常；
  • iDRAC 无控制器缓存告警；
  • 双电源或 UPS 供电稳定。

如果缓存保护异常，控制器可能自动降级到 Write Through，写入性能会明显下降。此时应修复硬件，而不是强制打开不安全的缓存策略。

七、iDRAC8 是什么

iDRAC 全称 Integrated Dell Remote Access Controller，是 Dell PowerEdge 内置的带外管理控制器。

它有独立处理器、内存、网络和固件。即使：

  • ESXi 没有安装；
  • 操作系统崩溃；
  • 服务器正在重启；
  • 系统盘损坏；

只要服务器接通电源并且 iDRAC 网络正常，管理员通常仍能远程访问它。

7.1 iDRAC8 能做什么

  • 查看 CPU、内存、磁盘、电源、风扇和温度；
  • 读取 System Event Log 和 Lifecycle Log；
  • 远程开机、关机、重启和强制断电；
  • 打开远程 KVM 控制台；
  • 远程挂载 ISO、光盘或 USB 镜像；
  • 修改一次性启动项；
  • 更新 BIOS、iDRAC、PERC 和网卡固件；
  • 导出 SupportAssist/TSR 诊断包；
  • 使用 RACADM、WS-Man、Redfish 等方式自动化管理。

7.2 iDRAC8 Express 与 Enterprise

R730 常见 iDRAC8 Express 或 Enterprise。

  • Express：基本硬件监控、日志、网络管理和电源控制；
  • Enterprise：通常增加完整远程控制台、虚拟介质、独立管理网络等高级能力。

十二、VMware ESXi 是什么

VMware ESXi 是 Type-1 裸金属虚拟化管理程序。它直接安装在物理服务器上，不需要先安装 Windows 或 Linux。

12.1 ESXi 中常见概念

名称 说明
VMkernel ESXi 的核心，负责 CPU、内存、存储和网络调度
Host Client 单机 Web 管理界面，地址通常为 https://ESXI-IP/ui
Datastore 存放虚拟机文件的逻辑存储空间
VMFS VMware 常用集群文件系统
vSwitch ESXi 内部虚拟交换机
vmnic ESXi 识别的物理网卡端口
vmk VMkernel 网络接口，用于管理、vMotion、存储等
Port Group 虚拟机或 VMkernel 接入虚拟交换机的逻辑网络
vCenter 集中管理多台 ESXi 的平台
vMotion 在主机之间迁移运行中的虚拟机，需要相应许可和配置
HA/DRS 高可用与资源调度功能，需要 vCenter 和相应许可

12.2 为什么 R730 常用 ESXi

  • 双路多核适合承载多台虚拟机；
  • 大内存适合数据库、Kubernetes 和中间件实验；
  • PERC RAID 和多盘位适合构建本地 Datastore；
  • iDRAC8 能远程安装和恢复；
  • 多网口方便拆分管理、存储和业务网络；
  • PCIe 扩展适合网卡、HBA 和 GPU 直通实验。

十三、R730 安装 ESXi 8 前的兼容性说明

这是全文最重要的一节。

13.1 能安装，不代表官方支持

Dell 官方 ESXi 7.x 兼容性矩阵明确列出了 R730/R730xd。对于 ESXi 8，R730 不应被默认视为官方认证平台。

安装前必须分别检查：

  1. Broadcom Compatibility Guide 中是否存在整机认证；
  2. CPU 是否被当前 ESXi 8 版本接受；
  3. PERC/HBA 驱动是否可用；
  4. 网卡驱动是否可用；
  5. 启动设备是否满足 ESXi 8 要求；
  6. 是否使用 Dell OEM Customized Image；
  7. 是否能接受未来升级失败、驱动移除和无厂商支持。

13.2 适合安装 ESXi 8 的情况

  • 家庭实验室；
  • 学习 VMware；
  • 非关键开发测试；
  • 有完整备份；
  • 可以随时重装；
  • 已验证网卡、PERC 和 CPU；
  • 接受非认证硬件风险。

13.3 不适合的情况

  • 核心生产数据库；
  • 只有一台主机、没有迁移和备份能力；
  • 必须获得 Dell/Broadcom 技术支持；
  • 依赖非常老或冷门的网卡/HBA；
  • 系统不能停机；
  • 数据只保存在本机 RAID 中。

13.4 ESXi 7.0 U3 是更保守的选择

如果目标是稳定而不是追新，R730 上的 ESXi 7.0 U3 通常更符合 Dell 官方兼容性资料。是否使用 ESXi 7 还要结合其安全维护状态、许可和组织策略，不应只看“能不能装”。

二十二、常见故障排查

22.1 ESXi 安装器找不到硬盘

可能原因：

  1. PERC 没有创建 Virtual Disk；
  2. Virtual Disk 没有初始化；
  3. 控制器处于不合适的模式；
  4. 使用了通用 ESXi ISO，缺少驱动；
  5. PERC 固件过旧；
  6. 阵列状态为 Foreign/Failed/Degraded；
  7. 选择了不被支持的 NVMe 转接方案。

处理：

  • 进入 F2 > Device Settings > PERC；
  • 确认 Virtual Disk 状态为 Ready/Optimal；
  • 使用 Dell Customized Image；
  • 更新 PERC 固件；
  • 检查物理盘和 CacheVault。

22.2 ESXi 提示 No Network Adapters

可能原因：

  • 网卡型号已从 ESXi 8 驱动中移除；
  • 使用通用 ISO；
  • NIC 固件太旧；
  • PCIe 插槽依赖 CPU2；
  • 插槽在 BIOS 中被禁用；
  • 第三方 USB 网卡不受支持。

处理：

  1. 使用 Dell OEM Customized Image；
  2. 在 Broadcom Compatibility Guide 查询具体 NIC；
  3. 更新网卡固件；
  4. 换到确认支持的 Intel/Broadcom 万兆网卡；
  5. 检查 CPU2、Riser 和 Slot Disablement。

不要把“网上有人成功”当作驱动支持证明。同型号网卡还可能有不同芯片、固件和 PCI ID。

22.3 iDRAC 虚拟介质无法启动

检查：

  • ISO 是否仍为 Mapped/Connected；
  • Next Boot 是否选择 Virtual CD/DVD；
  • Boot Mode 是否为 UEFI；
  • ISO 是否支持 UEFI；
  • 浏览器是否休眠；
  • iDRAC 许可证是否支持 Virtual Media；
  • iDRAC 固件是否过旧；
  • 是否能从 F11 手动看到虚拟光驱。

仍失败时改用 USB，不要与一条 100KB/s 的虚拟光驱死磕到凌晨三点。

22.4 安装完成后又进入安装器

原因通常是 ISO 仍然挂载，并且 Virtual CD/DVD 仍是第一启动项。

处理：

  1. 在 iDRAC 断开 Virtual Media；
  2. 进入 F11 选择 PERC Virtual Disk；
  3. 在 BIOS 调整 UEFI Boot Sequence；
  4. 确认 ESXi 安装盘被标记为可启动设备。

22.5 iDRAC 页面卡顿或无法打开控制台

处理顺序：

  1. 使用现代浏览器；
  2. 优先 HTML5 Console；
  3. 清理浏览器缓存；
  4. 检查 iDRAC 时间和证书；
  5. 更新 iDRAC8 固件；
  6. 执行：

软重置 iDRAC 通常不会重启服务器操作系统，但管理连接会暂时中断。

22.6 风扇转速异常高

可能原因：

  • 非 Dell 硬盘或 PCIe 设备；
  • GPU/HBA 需要更高散热；
  • 某个温度传感器异常；
  • 风扇缺失或故障；
  • 机箱盖未正确关闭；
  • BIOS/iDRAC 固件较旧；
  • Third-Party PCIe Card Cooling Response；
  • 入风口温度过高；
  • 某块卡插在不合适的槽位。

不建议通过 IPMI 命令长期强制把风扇压到极低转速。服务器安静了，芯片也可能被你安静地送走。

22.7 Secure Boot 无法开启

执行：

如果失败：

  • 找出未签名 VIB；
  • 移除不兼容驱动；
  • 使用 Dell/Broadcom 官方离线包；
  • 确认 ESXi 以 UEFI 模式安装；
  • 再进入 BIOS 开启 Secure Boot。

22.8 RAID 性能很差

检查：

  • Write Policy 是否降级为 Write Through；
  • CacheVault/电池是否异常；
  • 是否存在坏盘或重建；
  • 是否使用 SMR 盘；
  • RAID 级别是否适合随机写；
  • Datastore 是否接近写满；
  • SSD 是否为低耐久消费级型号；
  • PERC 固件是否过旧；
  • 是否存在磁盘介质错误。

22.9 ESXi 升级后网卡消失

ESXi 升级可能移除旧驱动。升级前：

  1. 查询目标版本 HCL；
  2. 导出当前 VIB 清单；
  3. 记录 vmnic 驱动和固件；
  4. 检查 Dell Addon/OEM Image；
  5. 备份主机配置；
  6. 确保虚拟机已有独立备份；
  7. 准备回滚或重装介质。

每周

  • 查看 iDRAC Overall Health；
  • 查看 PERC 和物理磁盘；
  • 查看最近硬件告警；
  • 确认备份任务成功；
  • 检查 Datastore 剩余空间。

每月

  • 导出重要配置；
  • 检查 ESXi 安全公告；
  • 检查 Dell 固件公告；
  • 抽查虚拟机恢复；
  • 检查 NTP 和证书；
  • 检查 UPS 和双电源状态。

每季度

  • 导出 TSR；
  • 检查内存可纠正错误趋势；
  • 检查 SSD/HDD 介质错误和寿命；
  • 清洁进风口和机房灰尘；
  • 验证灾难恢复流程；
  • 评估是否仍应继续运行非认证 ESXi 8。

每次升级前

  • 查整机、CPU、NIC、PERC 和存储 HCL；
  • 阅读 Release Notes 和 Known Issues；
  • 下载目标版本 ISO/Offline Bundle；
  • 备份 ESXi 配置和虚拟机；
  • 确认 iDRAC 可远程恢复；
  • 安排回滚方案；
  • 不在周五下班前做“顺手升级”。

二十五、最终验收清单

硬件

  • CPU、内存容量与采购配置一致；
  • 所有 DIMM 无 ECC 告警；
  • 所有风扇和电源正常；
  • PERC CacheVault/电池正常；
  • 物理磁盘无 Predictive Failure；
  • 双电源冗余正常；
  • BIOS、iDRAC、PERC、NIC 固件已记录。

BIOS

  • Boot Mode 为 UEFI；
  • Virtualization Technology 已开启；
  • Logical Processor 已开启；
  • Node Interleaving 已关闭；
  • USB 3.0 已按需开启；
  • GPU 场景已开启 MMIO Above 4GB；
  • ESXi 系统盘为第一启动项；
  • Secure Boot 状态符合当前驱动方案。

iDRAC

  • 默认密码已修改；
  • 静态 IP、DNS、NTP 已配置；
  • 管理 VLAN 已隔离；
  • 未暴露到公网；
  • Virtual Console 和 Virtual Media 可用；
  • 日志和 TSR 已导出；
  • iDRAC8 已更新至合适版本。

ESXi

  • 使用可信的 Dell Customized Image；
  • ESXi Build 已记录；
  • 管理 IP、DNS、Hostname 正常；
  • NTP 正常；
  • PERC 和 Datastore 正常；
  • 所有物理网卡正常；
  • SSH 默认关闭；
  • 许可证合法；
  • 主机配置已备份；
  • 虚拟机已有独立备份；
  • 已明确接受或规避 R730 + ESXi 8 的非认证风险。
